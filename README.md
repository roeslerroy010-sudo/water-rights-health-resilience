## 2. 我们的选题

### 为什么选水权交易

- **问题**：气候变化加剧水资源短缺，水污染导致健康负担上升；传统的水资源分配方式（行政配额、"价高者得"的自由市场）都无法有效应对健康风险。
- **切入点**：利用环境经济学工具（排污税、产权交易）在**水资源定价中内置健康外部性**，让市场自动将水分配到健康价值最高的用途。

### 和经济学背景的结合

- 团队两人均为 **北京大学人口、资源与环境经济学** 硕士生。
- 项目的核心逻辑完全建立在环境经济学理论之上：
  - **庇古税（Pigouvian Tax）**：对产生健康损害的水使用征税，使私人成本=社会成本
  - **科斯定理（Coase Theorem）**：清晰界定的水权 + 低交易成本 → 市场自发达到有效率配置
  - **激励相容（Incentive Compatibility）**：设计机制让每个用水户自觉遵守规则
- 这不是一个纯技术项目——它是一个 **经济学机制设计方案，辅以可视化交互工具**。

### 差异化优势

- **技术队不会做这个方向**：大多数参赛队伍的解决方案偏向传感器、水质监测、AI 预测等技术路线。我们的差异化在于 **用经济制度设计解决问题**，这在技术导向的 hackathon 中是稀缺的。
- **汇报时强调**：这是唯一一个从经济学出发的项目，我们的"技术"是**市场设计**。

## 3. 产品介绍

### 一句话

> 一个交互式决策支持工具，帮助流域管理者在气候不确定性下设计健康优先的水权市场。

### 核心经济学机制

两个核心工具，叠加在市场机制之上：

#### 1. 健康税（Health Tax, τ）

- 本质是 **庇古税**：对健康损害型用水（如工业排污）征收额外费用
- 定价公式：`P_trade = P_market + τ × H_loss × DALY(Q)`
  - `P_market`：市场出清价格（供需决定）
  - `τ`：健康税率（用户可调，0-50%）
  - `H_loss`：不同用水类型的健康损失系数（农业 0.25，工业 0.60，能源 0.05）
  - `DALY(Q)`：伤残调整生命年（WHO 标准）
- τ 越高，工业用水越贵，自动减少其用水量，水被重新分配到健康价值更高的用途

#### 2. 健康优先配额（Health Priority Quota）

- 总水量的一个百分比（默认 25%）**强制预留**给健康优先用户（居民饮用水、医院）
- 这部分水不参与市场交易，确保即使在高排放情景下，基本民生用水不受影响

#### 3. 激励相容检查

- 系统自动计算每个用水户的合规成本，判断是否有用户可能"违规"
- 绿色 `✓ Incentive Compatible` = 所有人都愿遵守规则
- 红色 `⚠ Violation` = 某些用户（如化工行业）违规动机强，需要调整参数

### 四个可调参数

| 参数 | 范围 | 默认值 | 作用 |
|------|------|--------|------|
| Health Tax Rate (τ) | 0% - 50% | 15% | 控制健康税强度 |
| Climate Scenario | SSP2-4.5 / SSP5-8.5 | SSP2-4.5 | 选择气候情景（水供应量不同） |
| Health Priority Quota | 10% - 40% | 25% | 设定健康优先用水比例 |
| Trading Cost | Low / Medium / High | Medium | 模拟不同交易摩擦成本 |

## 7. 常见 Q&A

预先准备好这些回答。如果评委不问，可以考虑在演讲中主动提及。

### Q1: 数据是真实的吗？

**回答要点：**
- 参数来源：中国水资源公报（流域水量）、CMIP6（气候情景）、WHO DALY 指南（健康影响系数）
- 流域设定：长江中游——这是一个**简化的示范案例**，不是实际校准的精确模型
- 核心目的是**演示机制逻辑**，不是做精确预测

### Q2: 能应用到真实流域吗？

**回答要点：**
- 可以。整个模型的核心数据在 `js/config.js` 中，替换其中的 `WATER_USERS`（用水户列表）、`BASIN_INFO`（流域参数）、`HEALTH_LOSS_COEFF`（健康损失系数）即可适配任何流域
- 实际应用需要：水文数据（水资源量）、用水户数据（位置、用水量、类型）、流行病学数据（健康基准值）

### Q3: 和现有水权市场有什么不同？

**回答要点：**
- 现有水权市场（如澳大利亚墨累达令、中国试点）的定价基础是 **经济效率**——水流向最高出价者
- 我们在价格中嵌入了 **健康外部性**——通过 τ × H_loss 项计算健康成本
- 简单说：传统市场让水从低价值用途流到高价值用途；我们让水从 **健康损害型用途** 流到 **健康促进型用途**

### Q4: 为什么用市场机制而不是命令控制（行政配额）？

**回答要点（核心论据）：**
- **动态性**：市场能自动响应水资源变化和价格信号；行政配额需要人工调整，反应慢
- **效率**：市场将水分配到边际价值最高的用途；静态配额做不到
- **激励相容**：我们内置了激励检查（绿色/红色状态），帮助管理者发现设计缺陷；命令控制难以事先预判合规困难
- 对经济学家来说，这个回答很有说服力

### Q5: 技术实现？

**回答要点：**
- 纯前端静态页面，HTML + CSS + JavaScript
- Leaflet.js 负责地图可视化，Chart.js 负责仪表盘图表
- 部署在 GitHub Pages，无需服务器、无需数据库
- 全场代码在 GitHub 上开源

### Q6: 只有两个人，时间这么短，为什么要做（挑战和收获）？

**回答要点（如果评委问团队背景）：**
- 我们是经济学背景，不是计算机专业
- 最大的挑战是把经济学逻辑转化为可交互的工具
- 收获：证明了经济学机制设计可以落地为可视化决策工具

### Q7: 健康税如何确定税率？会不会影响经济产出？

**回答要点：**
- 税率 τ 本身是可调的——这正是工具的价值：管理者可以在不同 τ 下比较健康收益和经济成本
- 默认 15% 是基于水媒疾病的 WHO 成本估算
- 工业成本上升可能导致部分产出下降，但工具右侧显示的 **Economic NPV** 已经包含了这种权衡——健康改善的经济价值通常会超过产出损失
- 经济学上说是"帕累托改进潜力"：将部分健康收益补偿受损方后，各方都可以变得更好

- 中期：与水利/卫生部门合作试点
- 长期：整合到国家水权交易平台（PPT Slide 11 有路线图）
